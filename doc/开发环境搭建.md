# NetNewsWire macOS 开发环境搭建

本文档面向 Swift/macOS 开发零基础用户，说明如何在本地 Mac 上搭建 NetNewsWire 的开发环境。

## 前提条件

- macOS (Apple Silicon)
- Homebrew 已安装
- 仅需 macOS 平台
- 采用 unsigned debug 构建，不需要 Apple ID 签名

## 第 1 步: 安装 Xcode

Command Line Tools 只能编译命令行程序，无法构建 macOS GUI 应用。必须安装完整的 Xcode。

1. 打开 Mac App Store，搜索 "Xcode"，点击安装
2. 安装完成后首次打开 Xcode，会提示安装额外组件，点击 "Install"
3. 将 xcode-select 指向 Xcode.app:
   ```bash
   sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
   ```
4. 验证:
   ```bash
   xcode-select -p
   # 应输出: /Applications/Xcode.app/Contents/Developer
   xcodebuild -version
   # 应输出 Xcode 版本号和 Build 号
   ```

## 第 2 步: 安装辅助工具

```bash
# xcbeautify - 格式化 xcodebuild 输出，项目的构建脚本 (build_and_test.sh) 依赖它
brew install xcbeautify

# swiftlint - Swift 代码风格检查
brew install swiftlint
```

不安装 xcbeautify 不影响直接用 `xcodebuild` 命令或 Xcode GUI 构建，但项目自带的 `build_and_test.sh` 脚本会报错。

## 第 3 步: 构建项目

在项目根目录执行 unsigned debug 构建:

```bash
xcodebuild -project NetNewsWire.xcodeproj \
  -scheme NetNewsWire \
  -configuration Debug \
  -destination "platform=macOS,arch=arm64" \
  CODE_SIGN_IDENTITY="" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO \
  SWIFT_TREAT_WARNINGS_AS_ERRORS=NO \
  build
```

不需要运行 `setup.sh` 或创建 `DeveloperSettings.xcconfig`，命令行参数已覆盖代码签名设置。

`SWIFT_TREAT_WARNINGS_AS_ERRORS=NO` 用于避免上游代码中预存在的 Swift type-check 超时警告（`-warn-long-function-bodies=300`）阻断构建。项目默认将所有 Swift 警告视为错误，而部分上游文件的函数 type-check 时间超过 300ms 限制，导致构建失败。此 flag 让这些警告恢复为警告，不影响构建结果。

### 已修复的构建问题: .nnwtheme 资源冲突

Xcode 26 使用 `PBXFileSystemSynchronizedRootGroup` 自动发现 `Shared/` 目录下的所有文件。`.nnwtheme` 主题包内的 `Info.plist`、`stylesheet.css`、`template.html` 会被展开复制到 `Resources/` 根目录，多个主题的同名文件产生冲突，导致 `error: Multiple commands produce` 错误。

修复方式：在 `project.pbxproj` 中将 `Shared` 的 `PBXFileSystemSynchronizedRootGroup` 的 `explicitFolders` 声明所有 `.nnwtheme` 目录，使 Xcode 将它们作为整体文件夹拷贝而非展开内容。此修复已提交到仓库，正常 clone 后构建不会遇到此问题。

## 第 4 步: 运行构建产物

```bash
# 找到构建产物
find ~/Library/Developer/Xcode/DerivedData/NetNewsWire-*/Build/Products/Debug \
  -name "NetNewsWire.app" -maxdepth 1

# 清除 quarantine 属性并直接运行
xattr -cr ~/Library/Developer/Xcode/DerivedData/NetNewsWire-*/Build/Products/Debug/NetNewsWire.app
open ~/Library/Developer/Xcode/DerivedData/NetNewsWire-*/Build/Products/Debug/NetNewsWire.app
```

如需复制到固定位置:

```bash
cp -R ~/Library/Developer/Xcode/DerivedData/NetNewsWire-*/Build/Products/Debug/NetNewsWire.app \
  /Applications/NetNewsWire-Dev.app
xattr -cr /Applications/NetNewsWire-Dev.app
open /Applications/NetNewsWire-Dev.app
```

## 第 5 步: 验证

1. **构建验证**: 第 3 步输出 `** BUILD SUCCEEDED **`
2. **运行验证**: 应用启动后能看到三栏界面 (侧边栏、文章列表、文章详情)
3. **测试验证**:
   ```bash
   xcodebuild -project NetNewsWire.xcodeproj \
     -scheme NetNewsWire \
     -destination "platform=macOS,arch=arm64" \
     CODE_SIGN_IDENTITY="" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO \
     SWIFT_TREAT_WARNINGS_AS_ERRORS=NO \
     test
   ```

## 日常开发工作流

修改代码后重复第 3 步构建即可。也可以用 Xcode GUI:

1. `open NetNewsWire.xcodeproj` 打开项目
2. 选择 scheme **NetNewsWire**，destination **My Mac**
3. 如遇签名错误: 选择项目 -> Signing & Capabilities -> 取消勾选 "Automatically manage signing"，或在 Build Settings 中将 Code Signing Identity 设为空

## 注意事项

- **Debug 构建的 Bundle ID 带 -DEBUG 后缀** (`com.ranchero.NetNewsWire-Evergreen-DEBUG`)，与 App Store 版使用不同 Bundle ID，可以共存不互相影响。
- **Unsigned 应用的 Keychain 限制**: unsigned 应用访问 Keychain 时可能弹出授权对话框。只使用本地 RSS 不登录同步服务账号的话不受影响。
- **核心功能完全可用**: RSS 订阅、抓取、阅读、本地账号管理等核心功能不依赖任何 API key。
